# The Ten AI Detection Signals

Ranked by severity. Every generation rule in the skill exists to prevent one or more of these.

## 1. Hallucinated Credentials (Career-Ending)

AI tools routinely invent certifications, assume programming language proficiency, or fabricate metrics. Multiple hiring managers state this is grounds for permanent disqualification.

**Prevention:** Hard constraint — only use data from the evidence bank. Never invent experience, metrics, tools, or certifications. Flag any generated content that doesn't trace back to a specific evidence bank entry.

## 2. Hidden Keyword Stuffing (Treated as Fraud)

White text, invisible sections, or dedicated keyword blocks. If discovered, treated as resume fraud by both ATS and human reviewers.

**Prevention:** Embed keywords through authentic achievement statements only. Never create keyword sections or force repetition.

## 3. Buzzword Summaries (Recognized Instantly)

"Results-driven professional with a proven track record of leveraging synergies." Recruiters see this hundreds of times per week.

**Prevention:** Hardcoded banned phrase list. See `banned-phrases.md`.

## 4. Generic Achievements (Most Common AI Tell)

"Led cross-functional teams to deliver projects on time" could describe anyone. GPTZero's hiring team: "If a resume bullet could describe almost anyone in a similar role and still be accurate, it's probably AI-generated."

**Prevention:** Evidence bank forces company-specific detail — tool names, team sizes, product names, internal systems.

## 5. Tone-Experience Mismatch (Collapses in Interviews)

Entry-level candidates claiming to have "architected end-to-end solutions" or "driven organizational transformation." These claims collapse under even light questioning.

**Prevention:** Calibrate language intensity to career level. Entry-level uses straightforward language. Senior uses proportionally stronger language. Never let language outpace experience.

## 6. Identical Sentence Structures (Algorithmically Flagged)

Every bullet opens with "Verb + object + result." AI detectors specifically flag how predictable sentence structure is.

**Prevention:** Vary sentence openings — some lead with result, some with context, some with action. Mix fragments and complete sentences. No more than 2 consecutive bullets starting with the same word class.

## 7. Robotic Phrasing (Fails Read-Aloud Test)

Phrasing no human would speak aloud. Overly formal constructions, excessive nominalization, awkward parallel structures.

**Prevention:** Natural language with deliberate imperfection. Industry-specific shorthand that only an insider would use. Apply the read-aloud test: would this person actually say this?

## 8. JD Mirroring (Flagged by Some ATS)

Copy-pasting job description language verbatim into the resume. Modern ATS tracks identical content.

**Prevention:** Paraphrase rather than echo JD language. Use the user's own vocabulary from the evidence bank to describe matching experience.

## 9. Keyword Overrepetition (43% of Recruiters Call It Dishonest)

Repeating the same skill or keyword five times across different bullets. Reads as gaming the system.

**Prevention:** Use each keyword 1-2 times in context. Rely on semantic variants and related terms rather than exact repetition.

## 10. Uniform Bullet Rhythm (Subtle but Cumulative)

Every bullet is the same length, same density, same grammatical pattern. Humans vary naturally.

**Prevention:** Mix one-line and two-line bullets. Vary information density. Some bullets are terse, others expand.
