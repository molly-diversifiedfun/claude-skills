**Step 1 — The Boundary Audit (7 questions, sequential)**

Ask one at a time. No batching.

> 1. "Do you pre-schedule specific, non-negotiable blocks for your project every week? YES / NO / SOMETIMES."
> 2. "When something 'urgent' comes up during project time, do you stop working? ALWAYS / SOMETIMES / NEVER."
> 3. "Do you allow notifications (email, Slack, texts) during project sessions? YES / NO / SOMETIMES."
> 4. "Have you communicated to others that your project time is off-limits? YES / NO / HAVEN'T TRIED."
> 5. "After a session, do you know exactly what you'll work on next time? YES / NO / SOMETIMES."
> 6. "Do you sacrifice project time for tasks that could genuinely wait? OFTEN / SOMETIMES / RARELY."
> 7. "Do you have a ritual to close project time, or do you just drift away? CLEAR CLOSE / I DRIFT."

Score:
- Q1 YES = 1; Q4 YES = 1; Q5 YES = 1
- Q2 NEVER = 1
- Q3 NO = 1
- Q6 RARELY = 1
- Q7 CLEAR CLOSE = 1

**Score 6-7 — 🟢 Strong foundation.** "Issue is scope or momentum, not time. Run `/unstuck scope` or `/unstuck audit` instead — this skill assumes you have time, you just need to use it."

**Score 3-5 — 🟡 Cracks in the walls.** "Continue. The plan below targets specific cracks."

**Score 0-2 — 🔴 Zero protection.** "Build defense before Day 1. Without this, the project won't ship."

---

**Step 2 — Name your #1 breaker (external + internal)**

Specific. Not "distractions."

**External (other people's demands):**

> "Examples of specific: 'Spouse walks in to chat during my 7-9pm block.' 'Manager pings on Slack at 8pm with non-urgent stuff.' 'Toddler wakes up at 5:30am, killing my pre-work block.' Not 'family' — 'spouse-during-7pm-block.'"

**Internal (own habits / avoidance):**

> "Examples of specific: 'I open Twitter when I hit the first hard problem.' 'I research alternatives instead of building.' 'I refactor working code instead of shipping new features.' 'I eat 20 min into the block reading email.' Not 'distractions' — name the actual avoidance pattern."

If the buyer goes generic ("distractions," "social media," "life"), push:

> "That's the category, not the breaker. The breaker is the SPECIFIC moment + behavior. What's the actual scene? Where are you sitting, what just happened, what did you do instead?"

Lock both. Add to artifact.

**Step 3 — Build 3 dedicated build blocks**

Pull from User Context Section A (work schedule, family context, peak energy). Propose 3 blocks sized to the buyer's reality. Each:

| Block | Day | Time | Location | Hours |
|---|---|---|---|---|
| 1 | | | | |
| 2 | | | | |
| 3 | | | | |

**Total weekly hours: ___**

Anti-pattern: 5-block plans with 60-min each. Better: 3 blocks at 90-120 min. Fewer blocks = less context-switching cost; longer blocks = real work happens.

Push back on aspirational schedules:

- "Saturday 6am" if buyer has a toddler waking at 5:30 → not a real block
- "Wednesday 9pm" if buyer reports peak energy at noon → mismatched
- "Whenever I have time" → not a block

**Step 3b — External defense (specific, not "headphones")**

For the locked external breaker, design the defense:

| External breaker | Wishful defense | Real defense |
|---|---|---|
| Spouse interrupts 7-9pm | "I'll wear headphones" | "Explicit agreement: door closed during 7-9pm = build time. They handle dinner cleanup. I handle Sunday cleanup in exchange." |
| Manager pings at 8pm | "I'll ignore notifications" | "Slack do-not-disturb 7pm-7am, with explicit Slack-status message: 'Heads-down, back tomorrow.' One-time conversation with manager about night-time pings." |
| Toddler wakes early | "I'll get up earlier" | "Block moves to 9-11pm post-bedtime instead. Morning block was a wish, not a plan." |

Force a structural change. If the buyer pushes back ("can't I just be more disciplined?"), respond:

> "Discipline isn't the fix. If it were, you'd already be disciplined. The fix is structural — what's the actual change to the environment / calendar / agreement that makes the breach impossible? Pick one."

**Step 3c — Internal defense (time-boxed, not "stay focused")**

For the locked internal breaker:

| Internal breaker | Wishful defense | Real defense |
|---|---|---|
| Research rabbit-hole | "I'll stay focused" | "10-min timer on any research question. When it goes off, go with what I have, even if it's incomplete." |
| Twitter at first hard problem | "Willpower" | "Phone in another room. Browser blocker (Cold Turkey / Freedom) blocks Twitter during build blocks." |
| Refactoring working code | "Avoid the urge" | "Hard rule: no editing files I haven't touched today. New feature first; refactor in a separate session." |

Time-box every defense. "Stay focused" has no edge condition; "10-min timer" does.

**Step 3d — Accountability / build partner**

Ask: "Who reviews your weekly progress? One human, by name. Not 'my followers' — one specific person."

Options:
- Existing friend who's also building
- Paid accountability ($497/mo Unstuck container, etc.)
- Forum / Slack community where you post weekly check-ins
- Build partner (Molly's flagship $4,500 6-week engagement)

If the buyer says "no one" → push:

> "Solo without accountability has ~5% ship rate in Molly's data. With one human reviewing your week, it's 60%+. The single biggest leverage point in this plan. Pick one before we move on."

**Step 4 — Lock the session protocol (Park Downhill)**

**🔌 Startup ritual (5 min, every session):**

- [ ] Phone on Do Not Disturb (and out of sight)
- [ ] Close all non-project browser tabs
- [ ] Review V1 outcome statement (from User Context D.2)
- [ ] Check "next action" from last session's Park Downhill
- [ ] START. Don't check email. Just start.

**🛑 Shutdown ritual (5 min, every session) — Park Downhill:**

- [ ] Save / commit work (even if messy)
- [ ] Write down tomorrow's EXACT first task
- [ ] Close all project tabs and windows
- [ ] One line in your log: "I showed up. That counts."

The Park Downhill habit is load-bearing. The next session's friction is mostly "what was I doing again?" Writing down tomorrow's first task tonight kills that friction.

Cite Newport (2016) on deep work + Wendy Wood (2019) on cue-based habit formation where it sharpens.
